## 问题：CE 遗漏了 JumpLifter

JumpLifter 定义了 `Mech_JumpLifter`，一种轻型机械体（body size 0.7），继承自 `LightMechanoid`。CE 对原版全部 6 种轻型机械体逐一做了补丁，但 JumpLifter 作为第三方 mod 添加的轻型机械体未被覆盖，导致两个关键系统缺失：

### 1. 缺少 `CompProperties_ArmorDurability` — 装甲永不退化

CE 为每种原版轻型机械体单独注入 `CompProperties_ArmorDurability`，赋予其装甲耐久值及自修复逻辑。JumpLifter 未被注入该 Comp，意味着它的装甲在受火力打击后 **永远不会退化**——这是一个隐性 buff，违反了 CE 的装甲耐久系统设计意图。

### 2. 缺少 `RacePropertiesExtensionCE` — bodyShape 未定义

CE 通过 `RacePropertiesExtensionCE` 的 `bodyShape` 字段为每种机械体指定碰撞体型。JumpLifter 缺失该扩展，运行时会触发：

```
ErrorOnce("CE returning BodyType Undefined for pawn ...")
```

该错误在机械体倒地时出现。更严重的是，bodyShape 缺失导致碰撞因子回退到 Invalid 体型的全 1.0 默认值，而非 Humanoid 体型正确的差异化数值：

| 姿态参数 | Invalid（错误回退值） | Humanoid（正确值） |
|----------|----------------------|-------------------|
| width（站立宽度） | 1.0 | 0.25 |
| height（站立高度） | 1.0 | 1.0 |
| widthLaying（倒地宽度） | 1.0 | 0.5 |
| heightLaying（倒地高度） | 1.0 | 0.3 |

站立时宽度被放大 4 倍，倒地时截面积从 0.15 膨胀到 1.0——弹道命中判定完全失真。

### 继承自父级的正常属性

以下属性通过 CE 对 `LightMechanoid` 父级的补丁正常继承，无需额外修补：

- 装甲值：Sharp 2, Blunt 3
- ToolCE 近战：head blunt, power 4, AP 0.5
- 近战属性：MeleeDodgeChance 0.1, MeleeCritChance 0.04, MeleeParryChance 0.04
- 负重：CarryWeight 50, CarryBulk 20

## 设计决策

### 耐久值选择：700

CE 对轻型机械体的耐久值分三档：

| 档位 | 耐久值 | 适用对象 | 依据 |
|------|--------|---------|------|
| 最小型 | 500 | Cleansweeper | 体型最小的轻型机械体 |
| **标准型** | **700** | **Militor, Lifter, Constructoid, Fabricor, Agrihand, Paramedic** | **使用 `Mech_Light` body (size 0.7) 的标准轻型机械体** |
| 重装型 | 1000 | WarUrchin | 体型最大、面向战斗的轻型机械体 |

JumpLifter 共享 `Mech_Light` body（size 0.7），与标准档的其他机械体完全一致。选择 700 是唯一符合 CE 分档逻辑的值。

### bodyShape 选择：Humanoid

CE 对轻型机械体的 bodyShape 按运动方式分类：

- **Humanoid**：双足行走机械体——Militor, Lifter, Fabricor, Paramedic
- **Vehicle**：轮式/履带式机械体——Constructoid, Cleansweeper, WarUrchin

JumpLifter 使用 `Mech_Light` body 配合 `SmallMechanicalLegs`（双足），归入 Humanoid 无争议。

### 不覆盖负重（保留父级 CarryWeight 50 / CarryBulk 20）

原版 Lifter 的 CE 补丁将负重提升至 CarryWeight 60 / CarryBulk 25，这是对其低移速（MoveSpeed 2.8）的补偿。JumpLifter 的定位截然不同：

- AutoBlink 能力：每 60 ticks 自动传送，最远 400 格
- MoveSpeed 5.2（Lifter 的近两倍）

传送 + 高移速已经使 JumpLifter 成为远优于 Lifter 的运输单位。保留父级的 50/20 维持了速度与容量之间的权衡关系。提升负重会使 JumpLifter 在运输效率上过度碾压 Lifter。

### 不覆盖战斗属性

JumpLifter 注册为 `NonCombatLightMechanoidKind`，无武器标签。父级继承的近战属性（MeleeDodgeChance 0.1 等）作为最后手段的自卫能力已经足够，无需调整。

## 再生与修复参数

```
Durability:       700
Regenerates:      true
RegenInterval:    1250 ticks
RegenValue:       5
→ 从零完全自修复: 700 / 5 * 1250 = 175,000 ticks (~48.6 小时)

Repairable:       true
RepairIngredients: Steel 5 + Plasteel 5
RepairTime:       300
RepairValue:      200
→ 每次手动修复恢复 200 点耐久, 需 4 次修满

MinArmorPct:      0.5
→ 装甲效能下限 50% (最低 Sharp 1.0, Blunt 1.5)
```

所有数值与 Militor、Lifter 等标准轻型机械体完全一致。

## 实现细节

补丁文件：`Patches/JumpLifter.xml`

### 门控结构

外层使用 `PatchOperationFindMod` 同时检测 "Combat Extended" 和 "JumpLifter" 两个 mod 是否加载，仅在两者同时存在时执行补丁。内层为 `PatchOperationSequence`，包含两个操作。

### 操作 1：注入 ArmorDurability Comp

```xml
<li Class="PatchOperationAdd">
    <xpath>Defs/ThingDef[defName="Mech_JumpLifter"]/comps</xpath>
    <value>
        <li Class="CombatExtended.CompProperties_ArmorDurability">
            <Durability>700</Durability>
            <Regenerates>true</Regenerates>
            <RegenInterval>1250</RegenInterval>
            <RegenValue>5</RegenValue>
            <Repairable>true</Repairable>
            <RepairIngredients>
                <Steel>5</Steel>
                <Plasteel>5</Plasteel>
            </RepairIngredients>
            <RepairTime>300</RepairTime>
            <RepairValue>200</RepairValue>
            <MinArmorPct>0.5</MinArmorPct>
        </li>
    </value>
</li>
```

使用 `PatchOperationAdd` 而非条件创建，因为 JumpLifter 的 ThingDef 已自带 `<comps>` 节点（包含其 AutoBlink 能力的 Comp），直接追加即可。

### 操作 2：注入 bodyShape 扩展

```xml
<li Class="PatchOperationAddModExtension">
    <xpath>Defs/ThingDef[defName="Mech_JumpLifter"]</xpath>
    <value>
        <li Class="CombatExtended.RacePropertiesExtensionCE">
            <bodyShape>Humanoid</bodyShape>
        </li>
    </value>
</li>
```

使用 `PatchOperationAddModExtension` 而非 `PatchOperationAdd`，该操作会在 `<modExtensions>` 节点不存在时自动创建，避免了额外的条件判断。

## 备选方案排除

**为什么不用耐久值 1000？**
WarUrchin 的 1000 对应战斗强化型轻型机械体。JumpLifter 是非战斗搬运单位，不符合该档位的定位。

**为什么不用耐久值 500？**
Cleansweeper 的 500 对应最小型轻型机械体变体。JumpLifter 使用标准 `Mech_Light` body（size 0.7），应归入 700 档。

**为什么不用 Vehicle bodyShape？**
Vehicle 专用于轮式/履带式机械体。JumpLifter 拥有双足 `SmallMechanicalLegs`，必须使用 Humanoid。

**为什么不像 Lifter 一样提升负重？**
Lifter 的 CarryWeight 60 / CarryBulk 25 是对其低移速（2.8）的补偿。JumpLifter 拥有 AutoBlink（60 ticks 间隔，400 格范围）和 MoveSpeed 5.2 双重优势，其单位时间运输效率已远超 Lifter。提升负重会打破权衡，使其成为无条件上位替代。
